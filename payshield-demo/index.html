Live Attack Simulation Rules

This tab shows the active rules configured in the **Requestly Browser Extension**. In a real attack, these rules run transparently in the browser intercepting the network request mid-flight. PayShield's Edge Gateway is designed to catch the modifications made by rules exactly like these.

Modify Headers Rule
ACTIVE
Target URL contains
/api/v1/payment/initiate
Action: Add/Modify Request Header
Key: X-Requestly-Modified
Value: true
This rule simulates an attacker injecting custom headers (like proxy markers or spoofed origins). PayShield's middleware scans for expected schemas and known tampering tags.
